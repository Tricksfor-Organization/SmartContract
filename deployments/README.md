# Deployments

This folder contains deployment records for the Tricksfor smart contracts on each network.

Each deployment record captures the addresses and constructor arguments used when a contract was deployed, enabling verification, reproducibility, and auditability.

## Structure

Deployment records are organised by network name using the `{chain}-{stage}` convention:

```text
deployments/
  config/
    ethereum-sepolia/
      deployment-params.json
    ethereum-mainnet/
      deployment-params.json
    polygon-amoy/
      deployment-params.json
    polygon-mainnet/
      deployment-params.json
    optimism-sepolia/
      deployment-params.json
    optimism-mainnet/
      deployment-params.json
    bsc-testnet/
      deployment-params.json
    bsc-mainnet/
      deployment-params.json
    avalanche-fuji/
      deployment-params.json
    avalanche-mainnet/
      deployment-params.json
  ethereum-sepolia/
  ethereum-mainnet/
  polygon-amoy/
  polygon-mainnet/
  optimism-sepolia/
  optimism-mainnet/
  bsc-testnet/
  bsc-mainnet/
  avalanche-fuji/
  avalanche-mainnet/
```

The `config/` subdirectory holds non-secret deployment parameters (NFT name, symbol, URIs, royalty configuration) that are committed to the repository. The network output directories are populated at deployment time.

Each network folder contains one JSON file per deployed contract:

```json
{
  "contractName": "TricksforBoosterNFT",
  "address": "0x...",
  "transactionHash": "0x...",
  "blockNumber": 12345678,
  "deployedAt": "2025-01-01T00:00:00Z",
  "constructorArgs": []
}
```

## Important Notes

- Do **not** commit private keys or mnemonics to this folder.
- ABI and BIN artifacts are stored in `/artifacts/contracts/` (generated by `npx hardhat compile`) — not here.
- The NFT contract is always deployed before the staking contract. The staking contract constructor requires the NFT contract address.
- The `config/` files contain only non-secret parameters. Secrets (`RPC_URL`, `DEPLOYER_PRIVATE_KEY`) are always stored as GitHub Environment secrets and never committed.
- See [`docs/github-environments-setup.md`](../docs/github-environments-setup.md) for GitHub configuration instructions.
